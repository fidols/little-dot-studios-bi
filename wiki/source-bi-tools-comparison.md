# Source: BI Tools Comparison — Tableau vs Looker vs Power BI

**Sources:** improvado.io, dataslayer.ai, domo.com, marlabs.com
**Ingested:** 2026-05-12

## The Three Main BI Tools

### Tableau
- **Strength:** Visual flexibility — business users can explore and visualize data on their own
- **Best for:** Ad hoc analysis, interactive dashboards, non-technical stakeholders
- **Integrations:** Salesforce, Hadoop, Google BigQuery, most databases
- **Pricing:** ~$70/user/month (Creator), $15/user/month (Viewer)
- **JD fit:** LDS JD mentions "Design and manage intuitive Tableau dashboards" — Tableau is explicitly named

### Looker (Google Cloud)
- **Strength:** Governance and consistency — uses LookML to define metrics centrally
- **Best for:** Enterprises that need "one source of truth" — everyone sees the same numbers
- **Model:** Data modeling as code (version-controlled)
- **Pricing:** Enterprise pricing, typically more expensive than Tableau
- **JD fit:** Not explicitly named in LDS JD but common at media tech companies

### Power BI (Microsoft)
- **Strength:** Deep integration with Microsoft stack (Excel, Azure, Teams)
- **Best for:** Companies already on Microsoft 365
- **Pricing:** ~$10/user/month — most affordable
- **JD fit:** Not named in LDS JD; LDS likely uses Google Workspace (YouTube is Google)

## Which Tool to Learn First for This Role
Tableau is explicitly mentioned in the JD. Prioritize it.

Secondary: Power BI is widely used and easy to learn if you know Excel well. Looker is more developer-oriented and less likely to be the first tool you touch at a ~80-person company.

## How These Tools Fit LDS's Workflow
| Use Case | Tool |
|---|---|
| Executive KPI dashboard | Tableau |
| Finance profitability reports | Tableau or Excel |
| Time tracking analysis | Export from Harvest/Toggl → Tableau |
| CRM pipeline reporting | Salesforce/HubSpot built-in or Tableau |
| Ad hoc analysis | Excel / Google Sheets |
