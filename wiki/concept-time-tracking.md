# Time Tracking Analytics

## What It Is
Time tracking in a production/agency context means logging hours spent by each team member on each project, task, or client. The data is used for:
- Billing clients accurately (billable vs. non-billable hours)
- Comparing actual hours to project estimates (budget vs. actuals)
- Identifying which projects/clients are profitable vs. unprofitable
- Informing future project pricing and staffing estimates

## Common Tools
- **Harvest** — popular in agencies, integrates with project management tools
- **Toggl Track** — lightweight, widely used
- **Clockify** — free tier available
- **Jira / Monday.com** — time logging built into PM tools
- **Replicon, Mavenlink** — enterprise options

## Key Metrics
| Metric | Definition |
|---|---|
| **Utilization Rate** | Billable hours / total available hours. Industry benchmark: 65–75% for agencies |
| **Realization Rate** | Billable hours actually invoiced / total billable hours logged |
| **Budget Variance** | Actual hours vs. estimated hours on a project |
| **Burn Rate** | How fast a project is consuming its allocated budget |
| **Overtime / Overrun** | Hours logged beyond the project budget |

## Common Problems the BI Analyst Solves
1. **Inconsistent logging** — team members not logging time, or logging inaccurately. Fix: dashboards that surface missing entries, reminders, manager review workflows
2. **Budget overruns caught too late** — by the time you know a project is over budget, it's too late. Fix: real-time burn rate alerts
3. **Underpriced new business** — proposals don't account for true time cost. Fix: historical actuals database to inform future estimates
4. **Resourcing bottlenecks** — one team is overloaded while another is underutilized. Fix: utilization dashboards by team/person

## Relevance to LDS Role
The JD explicitly states: "Own and manage the company's project time-tracking platform." This is a core ownership responsibility, not just an analytical one. Means the BI Analyst needs to:
- Ensure data quality (people logging correctly)
- Build reports on utilization and budget variance
- Feed the data to Finance for profitability analysis
